ARIA

Accessible Rich Internet Applications is a set of attributes that define ways to make web content and web applications more accessible to people with disabilities.

HTML Accessibility

It's always beneficial to write HTML code with accessibility in mind to provide the user with a good way to navigate and interact with your sites.

Semantic HTML

Semantic HTML means using correct HTML elements for their correct purpose as much as possible.

Examples of semantic elements: <form>, <table>, <article> - clearly defines content.

Examples of non-semantic elements: <div> and <span> - tells nothing about its content.

Headings

The proper use of headers can make your content more clear and coherent.

It is important to use headings to show the document structure and the relationships between different sections.

There are 6 heading tags:

Heading 1

- a single topic

Heading 2

- a subtopic

Heading 3

- details of a topic

Heading 4

- additional details of a topic
Heading 5
- table of contents/lists
Heading 6
- same as heading 5

Good Link Text

A link text should explain clearly what information the reader will get by clicking on that link.

Example

Examples of good links:

  • Find out more about the HTML language
  • Read more about how to eat healthy
  • Buy tickets to Mars here

Examples of bad links:

  • Click here
  • Read more..
  • Buy tickets to Mars here
